## How I Used AI During Development
I treated AI tools as a **collaborative implementation partner**. I didn’t rely on AI to make decisions — I used it to **speed up execution**, especially on UI and layout work.

## Where AI Helped the Most

### UI Implementation

I used AI heavily to convert design ideas into working React components:

* Careers page layout
* Job cards and filters
* Recruiter dashboard structure
* Section-based rendering
### Responsive Design
AI helped with:

* Adjusting layouts across breakpoints
* Fixing overflow issues
* Improving spacing and alignment
* Making sure the UI worked cleanly on both mobile and desktop

### Debugging & Fixes
I used AI to quickly resolve issues like:

* Layout breaking on smaller screens
* Scroll/overflow bugs
* State updates not reflecting in preview etc

## Prompting Style I Used

I believe that **clear, specific prompts give much better results**.
My approach was:

* Start with a clear goal (e.g., “mobile-first job listing UI”)
* Specify layout expectations
* Add constraints (clean UI, no clutter, reusable components)
* Refine with follow-up prompts after seeing results

## Use of Design Tools & Exploration
I also:

* Generated UI design ideas using ChatGPT and Gemini
* Explored multiple careers pages and ATS products for inspiration
* Studied how real platforms structure job listings and company branding

This helped me:
* Avoid building a generic UI
* Make better layout and UX decisions
* Think more like a product builder rather than just a developer

## Learning New Tools During the Project

This project pushed me to learn things I hadn’t used before so i learned fast using AI:

* **Supabase** (database + backend integration)
* Environment configuration for frontend/backend
* Deployment workflows (frontend + backend)

## What I Learned About Using AI

* Precise prompts give significantly better results than broad ones
* It’s much more effective to iterate than to expect perfect output in one try
* AI is best used as a **fast execution and refinement tool**, not as a replacement for thinking

## Final Thoughts

AI helped me move faster, but the key decisions
like layout structure, user flow, and overall product direction —
were always driven by my own understanding and iteration.

This project ended up being a mix of:
* Learning new tools
* Improving UI/UX thinking
* Practicing full-stack integration
* Using AI effectively as a development partner
